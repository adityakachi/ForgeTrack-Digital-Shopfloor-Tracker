# 🔧 Aditya’s Forging Shop Digital Tracker A lightweight, fully functional, single-page web application designed for digital operations, attendance reporting, and productivity tracking on the manufacturing shop floor. Built entirely using standard frontend technologies (HTML5, CSS3, JavaScript), this application runs completely in the browser without requiring any backend servers or complex installation. --- ## 🚀 Live Deployment on GitHub Pages The main application file has been structured as `index.html` so it can be instantly deployed to **GitHub Pages**. ### How to host this on GitHub: 1. Create a **new repository** on your GitHub account. 2. Upload the `index.html` file into the root folder of that repository. 3. In your GitHub repository, click on the **Settings** gear icon at the top menu. 4. On the left sidebar, scroll down to the *Code and automation* section and click on **Pages**. 5. Under *Build and deployment* -> *Source*, ensure it is set to **Deploy from a branch**. 6. Under *Branch*, switch the dropdown from *None* to **main** (or `master`) and keep the folder path as `/ (root)`. Click **Save**. 7. Wait 1 to 2 minutes. Refresh the settings page, and a link will appear at the top saying: *"Your site is live at..."* Click that link to launch your live tracker! --- ## 🌟 Key Application Features * **Dynamic Task Reporting:** Captures real-time logs for shop floor actions (`Forging`, `Trimming`, `Inspection`, `Machining`, `Packaging`). Includes an automatic Worker ID auto-filler mapped directly from a dynamic worker dropdown selection. * **Daily Attendance Logger:** Separate tracking mechanism to instantly log shop-floor presence (`Present`, `Absent`, `Leave`) mapped by worker IDs. * **Tool Life Dashboard & Maintenance Alerts:** Real-time visibility into machine tool cycles (`T2024A`, `T2024B`, `T2024C`). Integrates a critical warning threshold that automatically flags any tool with less than 20 cycles in red font with a replacement notice (⚠️). * **Interactive Data Analytics:** Powered by `Chart.js`, a real-time bar graph dynamically aggregates data on the fly to display a "Tasks per Day" summary as entries are submitted. * **Industrial Grade Spreadsheet Exports:** Powered by the `SheetJS` library, users can: * Export the complete data model (Combined Tasks + Attendance records) into a native Excel sheet (`ForgingShop_Combined_Report.xlsx`). * Use an interactive calendar month picker to generate focused monthly audit spreadsheets (`ForgingShop_Monthly_Report_YYYY-MM.xlsx`). * **Contextual AI Micro-Assistant:** Built-in simulation script logic reads current operator context and task status to instantly generate live safety tips and equipment instructions. * **Shop Safety & Compliance Cards:** Highly accessible documentation directly below the layout providing safety procedures (PPE, die crack checks, coolant flow, protective wrappers) across all major cells. --- ## 💾 Architecture & Data Persistence * **Zero Backend Costs:** Requires absolutely no database management systems or runtime environments (Node.js, Python, etc.). * **Local Storage Layer:** All entries from the Task and Attendance modules save securely inside the user's localized browser memory (`localStorage`). Data persists naturally across browser refreshes, tab closures, and device restarts. * **External Dependencies (CDN Integration):** * `Chart.js` — For executing clean dashboard visualization. * `SheetJS (XLSX)` — For compiling native, downloadable Excel binary matrices straight out of browser memory strings. --- ## 📝 Authorship & License * **Designed By:** Aditya * **Prototype Environment:** Aditya’s Forging Shop Digital Platform Project.
